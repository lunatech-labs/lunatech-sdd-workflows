  Technical approach (revised for work + break + env durations):
  - Thin stdlib server.js wrapping a single in-memory lib/session.js. Reads
  WORK_SECONDS/BREAK_SECONDS/PORT from env at startup, logs the bound port.
  - Two-phase lifecycle with a phase field: { remainingSeconds, totalSeconds, 
  phase, status, banner? }. Work runs → at zero sets the banner + switches to
  break (still running) → break zero = done. Start from done restarts.
  - Honest focus.js reuse, two-fold (AC6): per-second decrement in both phases
  driven by runInterval via a server-owned scheduler; MM:SS via formatTime; the
  banner text is captured by calling focus.js's notify with a capturing sink
  (not hardcoded). runSession can't be reused directly (no pause/query), so the
  server sequences two runIntervals itself.
  - Env durations with safe fallback (AC5b): pure parseDurations(env) validates
  (same ^\d+$ + >0 shape as focus.js), falls back to 1500/300 with a logged
  warning, never crashes.
  - Static UI shows phase + banner; container's compose.yaml sets the two
  duration env vars in one place.
